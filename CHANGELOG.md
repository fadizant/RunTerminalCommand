# Change Log

## 1.0.0

- Initial release

## 1.0.1

- Update documentation

## 1.0.2

- Added `{#opt:Label:option1,option2,...}` variable syntax — shows a dropdown picker when running a command
- Added **Settings** entry at the bottom of the command list (separated by a divider) to open `settings.json` and scroll directly to the `terminalCommandsRunner` section

## 1.0.3

- Added key/value syntax for dropdown options: `{#opt:Label:Display Name=value,...}` — the list shows the display name while the value is inserted into the command
- Updated default sample commands to Node.js examples showcasing all variable types (`{resource}`, `{#sym:...}`, `{#opt:...}`)
